Ever wondered what happens when you type an URL into your browser?

By : Kashyap Milind Kale

When you type facebook.com in your browser and click enter, In a few seconds you get to see , comment, like etc your friends post. But ever wondered what happens in those few seconds after you entered that url and before you get to see the content of facebook ?

Let us learn today what happens in those few seconds after you enter the url in your browser, It is also one of the hot favourite questions for software engineering interviews.

Let’s say you entered an URL into your browser. But wait hang on a minute! Let us first understand what an url actually is.

URL stands for Universal Resource Locator. It has four parts,

Scheme

  1. Scheme
  1. Domain
  1. Path
  1. Resource

consider the url given below as an example.

http://thementorhub.in/blog/tech/sysdesign
  • Here, “http://” is the scheme. Scheme tells the browser to connect with the server using the protocol “Http”.
  • Moving ahead “thementorhub.in” is called as domain, It is basically name of the site.
  • “/blog/tech” is the Path, And “/sysdesign” is the resource.
  • There is always a confusion between path and resource, Consider path as the directory and resource as the file in the filesystem.
  • Path and the resource together identifies the resource on the site we want to get.

Now that you know what an URL actually is, Let us continue where we had left. Once the URL is entered in the browser, Browser now has to discover how to connect with the server and this task is established by a process called DNS Lookup.

DNS stands for domain name system. It is also sometimes referred as the phonebook of the internet. DNS is a directory service that provides a mapping between the name of a host on the network and its numerical address also called as IP Addresses. To increase the speed of the DNS Lookup process the DNS information is cached in both browser and operating system. The browser stores it for a short period of time. If the dns cache is erased from the browser, Browser requests operating system for the cache information.

If you want to look for the cache stored in your windows os, Here is a quick command to search that in powershell.

Get-DnsClientCache

If the dns information in not found in the operating system cache then the browser makes an request to DNS Resolver. A DNS resolver the resolver is one part of a larger decentralized DNS When you send your request to the DNS resolver, the resolver accesses other servers in the DNS to obtain the address, then sends you the response. The information is cached at every step of the process.

Once the browser gets back the IP Address of the domain, It can now establish a TCP connection with the server. In case the protocol is “Https” it requires a process called as SSL/TLS handshake.

TCP/IP stands for Transmission Control Protocol/Internet Protocol and is a suite of communication protocols used to interconnect network devices on the internet.. TCP provides a full duplex connection between two machines, allowing them to exchange large volumes of data efficiently.

After the connection is successfully established the browser sends a http request to the server, The server then processes the request and sends back the http response to the browser.Finally browser renders the http response and you get to see the requested content!
